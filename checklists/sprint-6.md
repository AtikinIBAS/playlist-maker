# Sprint 6 Checklist

Самопроверка перед промежуточным ревью.

## Главный экран и навигация

- [x] Реализован главный экран приложения.
- [x] Настроена базовая навигация между экранами.
- [x] Переходы выполняются реально, без использования Toast вместо навигации.

## Экран настроек

- [x] Реализованы кнопки «Поделиться приложением», «Написать разработчикам», «Пользовательское соглашение».
- [x] Кнопка «Поделиться» открывает системный шэринг.
- [x] Кнопка «Написать разработчикам» открывает почтовый клиент с подставленными данными.
- [x] Кнопка «Пользовательское соглашение» открывает браузер.
- [x] Все строки вынесены в `strings.xml`, в коде нет лишнего хардкода.

## Экран поиска — UI

- [x] Реализовано поле ввода с placeholder «Поиск».
- [x] Добавлены иконка поиска и кнопка очистки.
- [x] При нажатии на «х» поле очищается, результаты скрываются.
- [x] Результаты отображаются с использованием `LazyColumn`.
- [x] Элементы списка кликабельны как элементы списка поиска.

## Поиск — данные и архитектура

- [x] Реализован `Storage` как эмулятор данных.
- [x] Реализован репозиторий поиска.
- [x] Поиск работает без учёта регистра.
- [x] Фильтрация выполняется в слое `Data/Domain`, а не в UI.
- [x] Длительность треков форматируется в формате `mm:ss`.
- [x] Проект разделён на слои `Data / Domain / UI`.

## Поиск — MVVM и состояния

- [x] Реализован `SearchViewModel`.
- [x] Добавлены состояния `Initial`, `Searching`, `Success`, `Fail`.
- [x] По нажатию на иконку лупы запускается поиск.
- [x] Во время поиска отображается индикатор загрузки.
- [x] При ошибке показывается сообщение, приложение не падает.
- [x] Используются `viewModelScope` и `Dispatchers.IO` для работы с данными.

## Общие требования

- [x] Приложение собирается и запускается без ошибок.
- [x] Реализованные экраны работают корректно.
- [x] Архитектура не нарушена, UI не обращается напрямую к слою Data.
